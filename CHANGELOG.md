# 更新日志

## v1.3.0 - 2026-04-23

### 架构优化

1. **知识库驱动的需求分析**
   - 强制读取业务知识库和代码知识库进行需求类型判断
   - 区分新增需求和调整需求：
     - 新增需求：依赖大模型自主理解
     - 调整需求：严格基于知识库现有内容进行拆解优化
   - 自动识别关联功能和模块，提供拆解参考

2. **智能追问机制**
   - 基于 AI 理解动态生成追问问题，而非预设固定问题
   - 支持需求类型确认流程：先识别再确认，确认后再拆解
   - 支持用户否认操作，自动清空已识别信息等待重新输入

### 功能改进

1. **导入流程优化**
   - 移除页面上的"跳过已填写"复选框
   - 上传文件后自动检测已填写内容
   - 智能追问用户是否覆盖已填写内容
   - 支持关键词识别："覆盖"、"跳过"、"是"、"否"等

2. **加载状态优化**
   - 将"加载中..."改为"Thinking..."样式
   - 添加动画圆点效果，提升用户体验

3. **会话上下文管理**
   - 修复会话 ID 传递问题，确保多轮对话连贯性
   - 所有响应包含 session_id 字段

4. **Excel 导入兼容性**
   - 支持多种工作表名称（需求拆解评估、工时评估表AI版本、需求清单等）
   - 自动检测并使用第一个可用工作表

### 代码优化

1. **`agent/dialog_manager.py`**
   - 强制知识库分析优先于规则匹配
   - 新增 `kb_analysis` 和 `related_features` 字段
   - 增强需求类型识别逻辑

2. **`agent/worktime_agent.py`**
   - 在处理每个需求前强制调用知识库分析
   - 将分析上下文传递给 LangGraph

3. **`app.py`**
   - 增强确认消息，显示知识库分析来源
   - 调整需求显示关联功能列表供用户参考

4. **`excel/reader.py`**
   - 添加多工作表名称支持
   - 自动检测可用工作表

5. **`templates/index.html`**
   - 添加 Thinking... 加载状态样式
   - 实现文件上传后的智能追问逻辑
   - 移除跳过已填写复选框

## v1.2.0 - 2026-04-22

### 架构重构

1. **前后端职责分离**
   - 后端统一响应格式：`{success, output{type, content}, meta{intent, stage}}`
   - 前端仅负责渲染，不再处理业务逻辑
   - 输出类型：`text`（文本消息）、`evaluation`（评估结果）、`thinking`（思考状态）、`error`（错误）

2. **意图识别系统**
   - 新增三种意图：`chat`（闲聊）、`new_task`（新需求）、`revise_task`（订正重跑）
   - 闲聊消息自动引导用户回到业务主题
   - revise_task 支持全流程重算

### 功能改进

1. **Thinking 状态输出**
   - 模型处理过程中输出"正在分析..."等状态信息
   - 支持逐步加载，提升用户体验

2. **流式响应接口**
   - 新增 `/chat/stream` 接口，支持 Server-Sent Events
   - 实时推送意图识别、思考状态、完成结果

3. **闲聊回复优化**
   - 扩展闲聊关键词库（问候、感谢、告别、自我介绍等）
   - 所有闲聊回复均包含引导提示，引导用户描述产品需求

### 问题修复

1. **闲聊消息处理**：修复前端未处理 `stage === 'chat'` 的问题
2. **反馈重新评估**：修复"不对"等反馈未触发重新评估的问题
3. **上下文混淆**：修复闲聊消息混入评估上下文的问题

### 代码优化

1. **`app.py`**
   - 新增 `make_response()` 统一响应格式生成器
   - `/chat` 接口全面使用统一格式
   - 修复流式接口闲聊回复不完整的问题

2. **`agent/dialog_manager.py`**
   - `analyze_intent()` 输出固定 JSON 格式
   - 调整意图识别优先级：question > evaluation

3. **`templates/index.html`**
   - 简化响应处理逻辑，仅根据 `output.type` 渲染
   - 添加 chat 状态处理
   - 支持统一响应格式和向下兼容

## v1.1.0 - 2026-04-14

### 功能改进

1. **智能需求识别优化**
   - 扩展业务模块关键词库：从10个模块扩展到13个（新增内容管理、搜索功能、移动端、接口开发）
   - 扩展需求类型关键词库：从3种类型扩展到5种（新增重构、技术债务）
   - 增加页面级关键词（如"登录页"、"订单列表"）提高识别准确度

2. **简化交互流程**
   - 实现"输入即评估"模式：用户输入需求后直接评估，无需多轮追问
   - 模块未识别时自动使用默认值"未指定模块"
   - 类型未识别时自动使用默认值"新增功能"
   - 在评估结果中显示智能识别状态和注意事项

3. **智能提示机制**
   - 自动识别成功时显示"【智能识别】已识别模块/类型"
   - 识别失败时显示警告提示并引导用户修改

### 问题修复

1. **前端 JavaScript 错误**：修复 `initWelcomeMessage is not defined` 错误（函数定义位置错误）
2. **后端 500 错误**：修复 `knowledge_manager.py` 缺少 `import time` 的问题

### 代码优化

1. **`agent/dialog_manager.py`**
   - 增强 `module_keywords` 和 `type_keywords` 关键词库
   - 修改 `check_info_complete()` 方法，支持自动补充默认值

2. **`app.py`**
   - 修改 `/chat` 接口，实现智能识别+直接评估模式
   - 添加智能识别结果显示和注意事项提示

3. **`templates/index.html`**
   - 修复 `initWelcomeMessage` 函数定义位置问题

4. **`agent/knowledge_manager.py`**
   - 添加缺失的 `import time` 语句
