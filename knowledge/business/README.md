# 业务知识库

将你的业务领域文档放在这个目录，AI 在拆解需求时会自动检索匹配的文档并注入 prompt。

## 文件格式

**Markdown（推荐）**，每个文件对应一个业务领域或子模块。

## 文件命名建议

以业务域命名，方便关键词匹配：
- `order_management.md`   — 订单管理业务规则
- `fulfillment_center.md` — 履约中台业务流程
- `inventory.md`          — 库存管理说明
- `user_permission.md`    — 用户权限体系

## 推荐的文档结构

```markdown
# 模块名称

## 业务背景
简要说明这个模块的业务目标。

## 核心流程
1. 步骤一...
2. 步骤二...

## 已有功能
- 功能A：说明
- 功能B：说明

## 业务规则
- 规则1
- 规则2

## 关键术语
- 术语A：定义
- 术语B：定义
```

## 注意事项

- 文件名中的关键词会用于匹配需求，命名要贴近业务语言
- 每个文件建议聚焦一个领域，不要把所有文档混在一个文件里
- 修改后立即生效，无需重启服务
- 也可以在 `.env` 中设置 `BUSINESS_KB_DIR` 指向外部目录，无需把文件复制到这里
